# Protocolo de Criatividade Combinatória

> **Severidade:** SHOULD | **Aplica-se a:** Agentes de criação (content, copywriter, creative-director, offer-engineer)
> **Origem:** Cross-pollination de domínios — Segunda-feira v6.2

## Princípio

Verdadeira criatividade não é inventar do zero — é combinar padrões conhecidos de formas inesperadas. Antes de propor qualquer solução criativa, o agente deve explorar pelo menos UMA perspectiva de outro domínio.

**Regra de ouro:** "A melhor ideia para seu domínio provavelmente já existe em outro domínio."

---

## Protocolo de 3 Etapas (obrigatório para criações importantes)

### Etapa 1: CONVENCIONAL
> Qual é a solução óbvia no meu domínio?

O agente propõe a solução que qualquer profissional da área proporia. Essa é a baseline.

### Etapa 2: ANALOGIA
> Como outro domínio resolve problema similar?

O agente consulta PELO MENOS 1 domínio diferente:

| Se você é... | Consulte... | Para obter... |
|-------------|------------|---------------|
| @content | @cold-outreach | Hooks de abertura de conversa |
| @content | @offer-engineer | Estrutura de stack de valor |
| @copywriter | @growth-hacker | Patterns de retenção de algoritmo |
| @creative-director | @ux-design-expert | Hierarquia visual e usabilidade |
| @offer-engineer | @swarm-simulator | Reação provável do mercado |
| @traffic | @content | Ângulos narrativos para criativos |
| @launch-strategist | @growth-hacker | Mecânicas de viralização |

### Etapa 3: INVERSÃO
> E se fizéssemos o oposto do convencional?

O agente propõe UMA alternativa que contradiz a solução óbvia:
- Se todo mundo faz longo → testa curto
- Se todo mundo é formal → testa irreverente
- Se todo mundo mostra resultado → mostra processo
- Se todo mundo vende → educa

## Quando Aplicar

| Situação | Aplicar protocolo? |
|----------|-------------------|
| Campanha nova (ângulo/criativo) | SIM — completo (3 etapas) |
| Post semanal recorrente | PARCIAL — etapa 1 + 2 |
| Resposta rápida/operacional | NÃO |
| Oferta ou lançamento | SIM — completo (3 etapas) |
| Bug fix ou tarefa técnica | NÃO |

## Output do Protocolo

Quando aplicado, incluir no output:

```markdown
### 💡 Processo Criativo
- **Convencional:** [solução óbvia]
- **Analogia:** [domínio consultado] → [insight extraído]
- **Inversão:** [alternativa contrária]
- **Escolha final:** [qual abordagem e por quê]
```

## Pattern Library como Combustível

Antes de criar, SEMPRE consultar ~/patterns/:
- `angles.md` → ângulos que já converteram
- `hooks.md` → aberturas que já funcionaram
- `formats.md` → formatos que já performaram
- `offers.md` → estruturas que já venderam

Esses padrões são a matéria-prima da combinação criativa.

---

## Anti-Patterns

| Evitar | Por quê |
|--------|---------|
| Criatividade pela criatividade | Se o convencional funciona, use. Não reinvente sem motivo. |
| Ignorar dados de performance | Criatividade sem dados é achismo. |
| Copiar sem adaptar | Analogia é adaptação, não cópia. Contexto importa. |
| Inversão forçada | Se a inversão não faz sentido, descarte. Não force. |
